# 위치 변수 3단계 분류 설계 (도시/준도시/도시 외곽)

작성: 2026-07-11. 사용자 승인 완료.

## 목적
대학 위치를 모르는 사용자가 캠퍼스가 도시인지 외곽인지 즉시 알 수 있도록,
기존 4단계 AI 서술(도심/도심 인접/외곽/교외·소도시, 141개가 "도심 인접"에 몰림 + `|` 복사 버그 18건)을
3단계 라벨 + 근거 서술로 교체한다.

## 분류 기준 (사용자 확정)
- **도시**: 대도시 도심 한복판 (도보권)
- **준도시**: 도심까지 대중교통 1시간 미만 (예: 나고야대 — 나고야역에서 지하철 약 20분), 또는 소도시/대학도시의 시내
- **도시 외곽**: 가장 가까운 도시 도심까지 1시간 이상이고 소도시 중심도 아닌 경우
- 소도시(인구 5~20만 대학도시 등) 중심 캠퍼스는 **준도시** (사용자 답변)

## 방식: B안 — Solar는 사실만 추정, 분류는 코드가
Solar 반환 JSON:
- `city`: 캠퍼스 소재 도시명
- `city_size`: `대도시`(광역권 약 50만+ 또는 국가 주요 도시) | `소도시`(약 5만~50만, 대학도시 포함) | `마을`(그 미만·시골)
- `commute_min`: 그 도시 도심까지 대중교통 소요시간(분, 정수. 도심 안이면 0)
- `transport`: 주 교통수단 (지하철/버스/트램 등)
- `major_city_note`: 소도시·마을일 때 가장 가까운 대도시와 대략 소요시간, 대도시면 ""

검증: `city_size`가 허용값 3개 중 하나가 아니거나 `|` 포함, 또는 `commute_min`이 숫자가 아니면
교정 메시지로 1회 재시도. 그래도 실패하면 `확인필요`.

## 분류 규칙 (location.py의 classify_tier)
| city_size | commute_min | tier |
|---|---|---|
| 대도시 | ≤ 10 | 도시 |
| 대도시 | 11 ~ 59 | 준도시 |
| 대도시 | ≥ 60 | 도시 외곽 |
| 소도시 | ≤ 30 | 준도시 |
| 소도시 | > 30 | 도시 외곽 |
| 마을 | 무관 | 도시 외곽 |

## 표시 형식
`location.value` = `"{tier} — {근거}"`. 예:
- `도시 — 멜버른 도심에 위치`
- `준도시 — 나고야 도심까지 지하철 약 20분`
- `준도시 — 소도시 Uppsala 시내 (스톡홀름에서 기차 약 40분)`
- `도시 외곽 — 가장 가까운 도시까지 버스 1시간 이상`

`location` 객체에 `tier` 필드 추가 저장 (사이트 필터용 여지). 출처 라벨은 기존대로 `AI추정`.

## 실행
크롤/추출 재실행 없음. 새 스크립트 `relocate.py`가 data.json 221개교의 위치만 재추정 →
data.json 갱신(중간 저장) → `build_table.py` + `build_site.py` 재실행.

## 검증
- classify_tier 단위 테스트 (경계값 10/11, 59/60, 30/31)
- 소량(5개교) 시험 후 전체 실행
- 3단계 분포 출력, 스팟체크: 나고야대→준도시, 멜버른대→도시, 미국 대학도시→준도시
- `|` 포함 setting 0건 확인
